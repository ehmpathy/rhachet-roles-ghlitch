# S3 Bucket Observability Provisions

## Goal
Enable efficient monitoring and optimization of S3 storage by:
- Visualizing bucket contents (tree structure with sizes)
- Detecting access patterns (last-used tracking)
- Identifying storage tier optimization opportunities

## Minimum-Cost Observability Stack

### 1. S3 Inventory (Core Foundation)
**Purpose**: Generate daily/weekly manifests of all objects with metadata
**Cost**: ~$0.0025 per million objects listed
**Provides**:
- Object key (path)
- Size
- Last modified date
- ETag
- Storage class
- Encryption status

**Setup**:
```bash
aws s3api put-bucket-inventory-configuration \
  --bucket SOURCE_BUCKET \
  --id daily-inventory \
  --inventory-configuration '{
    "Destination": {
      "S3BucketDestination": {
        "Bucket": "arn:aws:s3:::INVENTORY_BUCKET",
        "Format": "CSV",
        "Prefix": "inventory/"
      }
    },
    "IsEnabled": true,
    "Id": "daily-inventory",
    "IncludedObjectVersions": "Current",
    "OptionalFields": [
      "Size",
      "LastModifiedDate",
      "StorageClass",
      "IntelligentTieringAccessTier"
    ],
    "Schedule": {
      "Frequency": "Daily"
    }
  }'
```

**Analysis**: Process CSV files locally or with Athena to build tree structures and size aggregations

### 2. S3 Storage Class Analysis (Free)
**Purpose**: Automated access pattern analysis with storage class recommendations
**Cost**: FREE
**Provides**:
- Object age distribution
- Access pattern analysis (frequent vs infrequent)
- Automatic recommendations for lifecycle transitions
- Visualization in S3 Console

**Setup**:
```bash
aws s3api put-bucket-analytics-configuration \
  --bucket BUCKET_NAME \
  --id entire-bucket-analysis \
  --analytics-configuration '{
    "Id": "entire-bucket-analysis",
    "StorageClassAnalysis": {
      "DataExport": {
        "OutputSchemaVersion": "V_1",
        "Destination": {
          "S3BucketDestination": {
            "Format": "CSV",
            "Bucket": "arn:aws:s3:::ANALYTICS_BUCKET",
            "Prefix": "analytics/"
          }
        }
      }
    }
  }'
```

**Notes**:
- Takes 30 days to generate first report
- Updates daily after initial period
- Can create multiple configurations with filters for different prefixes

### 3. S3 Storage Lens (Free Tier)
**Purpose**: Organization-wide metrics and trends
**Cost**: FREE (basic metrics), ~$0.20 per million objects (advanced metrics)
**Provides**:
- Storage bytes by bucket/prefix
- Object count trends
- Storage class distribution
- Cost efficiency metrics (free tier)
- Advanced: Access patterns, activity metrics (paid)

**Recommendation**: Use **free tier only** for cost efficiency

**Setup**: Enable via S3 Console → Storage Lens → Default dashboard

### 4. CloudWatch Storage Metrics (Free)
**Purpose**: Basic storage volume monitoring
**Cost**: FREE (storage metrics), ~$0.30/metric/month (request metrics)
**Provides**:
- Daily storage bytes
- Number of objects
- Per-storage-class breakdown

**Recommendation**: Use **storage metrics only** (free), avoid request metrics

## Access Pattern Detection Strategy

Since S3 doesn't natively track "last accessed" (only last modified), use this approach:

### Option A: S3 Storage Class Analysis (Recommended)
- Enable Storage Class Analysis (free)
- Analyzes actual access logs internally
- Provides recommendations after 30 days
- No additional access logging costs

### Option B: S3 Intelligent-Tiering Access Tracking
- Move objects to Intelligent-Tiering
- Monitor which tier objects land in via Inventory reports
- `IntelligentTieringAccessTier` field shows: FREQUENT, INFREQUENT, ARCHIVE_INSTANT, ARCHIVE, DEEP_ARCHIVE
- This reveals actual access patterns without logging costs

### Option C: Server Access Logging (Expensive - Avoid Unless Necessary)
- Generates detailed logs for every request
- Storage costs for logs
- Processing costs (Athena queries)
- Only use if detailed request auditing required

## Recommended Minimum-Cost Configuration

```bash
# 1. Enable S3 Inventory (daily)
aws s3api put-bucket-inventory-configuration ...

# 2. Enable Storage Class Analysis (free, whole bucket)
aws s3api put-bucket-analytics-configuration ...

# 3. Use S3 Storage Lens free tier (already enabled by default)

# 4. For access pattern detection without logging:
#    Move candidates to Intelligent-Tiering and monitor via Inventory
```

## Analysis Workflow

### Tree Structure with Sizes
```bash
# Download latest inventory CSV
aws s3 cp s3://INVENTORY_BUCKET/inventory/latest/manifest.json .

# Process with local script or Athena query
# Build directory tree by aggregating sizes per prefix
```

### Access Pattern Detection
```bash
# Method 1: Check Storage Class Analysis reports (after 30 days)
aws s3 ls s3://ANALYTICS_BUCKET/analytics/

# Method 2: Query Intelligent-Tiering tiers from Inventory
# Objects in ARCHIVE tiers = not accessed in 90+ days
# Objects in INFREQUENT tier = not accessed in 30+ days
```

### Optimization Opportunities
1. **Review Storage Class Analysis recommendations** → shows which prefixes to move to IA/Glacier
2. **Query Inventory for objects in Standard class > 30 days old** → candidates for IA
3. **Query Inventory for objects in Standard class > 90 days old** → candidates for Glacier
4. **Check Intelligent-Tiering distribution** → validate if IT is working efficiently

## Cost Summary

| Feature | Cost | Value |
|---------|------|-------|
| S3 Inventory | ~$0.0025/M objects | Essential for tree structure & metadata |
| Storage Class Analysis | FREE | Best for access pattern recommendations |
| Storage Lens (free tier) | FREE | Organization-wide trends |
| CloudWatch Storage Metrics | FREE | Basic monitoring |
| **Total Monthly Cost** | **~$0.25 for 100M objects** | Complete observability |

**Costs to Avoid**:
- Server Access Logging: $0.002 per 10,000 logs + storage + query costs
- CloudWatch Request Metrics: $0.30 per metric/month
- Storage Lens Advanced: $0.20 per million objects/month

## Implementation Priority

1. **Week 1**: Enable S3 Inventory → immediate tree structure visibility
2. **Week 1**: Enable Storage Class Analysis → start 30-day observation period
3. **Week 2**: Build inventory processing scripts (tree aggregation)
4. **Week 5**: Review Storage Class Analysis recommendations
5. **Ongoing**: Query inventory monthly for optimization opportunities

## Key Insights

- **S3 Inventory** is the foundation for understanding "what's there"
- **Storage Class Analysis** is the free solution for "how it's accessed"
- **Intelligent-Tiering + Inventory** reveals access patterns without logging
- **Avoid access logging** unless audit requirements demand it
- **Total cost is negligible** compared to storage optimization savings
