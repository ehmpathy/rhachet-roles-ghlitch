# s3 metadata tables overview

## what it is (and why you’d use it)
- s3 automatically captures object metadata (system metadata, tags + user-defined metadata, and change events) from a **general-purpose bucket** and stores it in **read-only apache iceberg tables** you can query (two tables: **journal** for changes, **live inventory** for latest state).
- near real-time updates; great for discovery, compliance, ops, and ML data prep.

---

## release + availability
- **announced (preview):** dec 3, 2024 (re:invent)
- **ga:** jan 27, 2025 (initially us-east-1/2 and us-west-2)
- **mid-2025 upgrade:** support for **existing objects** (one-time backfill) + **price reduction (~33%)**
- **region expansion:** added more regions oct 22, 2025

---

## pricing
| feature | cost | notes |
|----------|------|-------|
| journal table | $0.30 per 1M updates | PUT/DELETE events |
| live inventory backfill | $0.30 per 1M existing objects | one-time load |
| live inventory monthly | $0.10 per 1M objects/month | only if bucket >1B objects |
| query costs | normal athena/redshift/emr charges | billed separately |

## how it works (query path)
- tables live in an **aws-managed s3 table bucket** (namespace like `b_<your-bucket>` with `journal` and `inventory` tables).
- integrate with **glue data catalog / lake formation**, query via **athena, emr, redshift, spark/trino**.

---

## how to activate

### option a — console (“by hand”)
1. open **s3 console → general purpose buckets → your bucket**
2. go to **metadata tables** and **create configuration**
3. choose **journal** record retention (enable + days) and optionally enable **live inventory** (triggers **backfill** of existing objects)

> backfill can take minutes to hours depending on object count. after backfill, updates appear in live inventory within ~1 hour.

---

### option b — cli / api / sdk

#### aws cli v2
\`\`\`bash
aws s3api create-bucket-metadata-configuration \
  --bucket my-bucket \
  --metadata-configuration '{
    "JournalTableConfiguration": {
      "RecordExpiration": { "Expiration": "ENABLED", "Days": 90 },
      "EncryptionConfiguration": { "SseAlgorithm": "AES256" }
    },
    "InventoryTableConfiguration": {
      "ConfigurationState": "ENABLED",
      "EncryptionConfiguration": { "SseAlgorithm": "AES256" }
    }
  }'
\`\`\`

#### aws sdk for javascript (v3 / typescript)
\`\`\`ts
import { S3Client, CreateBucketMetadataConfigurationCommand } from "@aws-sdk/client-s3";

const s3 = new S3Client({ region: "us-east-1" });

await s3.send(
  new CreateBucketMetadataConfigurationCommand({
    Bucket: "my-bucket",
    MetadataConfiguration: {
      JournalTableConfiguration: {
        RecordExpiration: { Expiration: "ENABLED", Days: 90 },
        EncryptionConfiguration: { SseAlgorithm: "AES256" }
      },
      InventoryTableConfiguration: {
        ConfigurationState: "ENABLED",
        EncryptionConfiguration: { SseAlgorithm: "AES256" }
      }
    }
  })
);
\`\`\`

**required permissions:**
\`\`\`
s3:CreateBucketMetadataTableConfiguration
s3tables:CreateTableBucket
s3tables:CreateNamespace
s3tables:CreateTable
s3tables:PutTablePolicy
s3tables:PutTableEncryption
\`\`\`

> use `kms` permissions if you encrypt with a custom key.

#### legacy v1 api (deprecated)
- old `create-bucket-metadata-table-configuration` replaced by v2.
- recreate configs (if made before jul 15, 2025) to gain expiration + inventory support.
