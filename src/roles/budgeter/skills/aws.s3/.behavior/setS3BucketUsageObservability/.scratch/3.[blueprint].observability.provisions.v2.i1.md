# AWS S3 Metadata Provisioning Blueprint

## Overview

This document describes the complete provisioning process for enabling AWS S3 Metadata on an existing S3 bucket, allowing object metadata to be queried through Amazon Athena.

**What is S3 Metadata?**
S3 Metadata accelerates data discovery by automatically capturing metadata for objects in general purpose buckets. The service stores this information in read-only Apache Iceberg tables that can be queried using standard SQL tools like Amazon Athena, Amazon EMR, Amazon Redshift, Apache Spark, and Trino.

## Prerequisites

### 1. Regional Availability
Your bucket must be located in one of these regions:
- US East (N. Virginia) - `us-east-1`
- US East (Ohio) - `us-east-2`
- US West (Oregon) - `us-west-2`

### 2. Bucket Type
- Must be a general purpose S3 bucket
- Cannot apply configuration at the prefix level (bucket-level only)

### 3. IAM Permissions Required

#### For Creating/Managing Metadata Configuration:

```json
{
  "Version": "2012-10-17",
  "Statement": [{
    "Sid": "PermissionsToWorkWithMetadataTables",
    "Effect": "Allow",
    "Action": [
      "s3:CreateBucketMetadataTableConfiguration",
      "s3:GetBucketMetadataTableConfiguration",
      "s3:DeleteBucketMetadataTableConfiguration",
      "s3:UpdateBucketMetadataJournalTableConfiguration",
      "s3:UpdateBucketMetadataInventoryTableConfiguration",
      "s3tables:CreateTableBucket",
      "s3tables:CreateNamespace",
      "s3tables:CreateTable",
      "s3tables:GetTable",
      "s3tables:PutTablePolicy",
      "s3tables:PutTableEncryption",
      "kms:DescribeKey"
    ],
    "Resource": [
      "arn:aws:s3:::YOUR-BUCKET-NAME",
      "arn:aws:s3tables:REGION:ACCOUNT-ID:bucket/aws-s3",
      "arn:aws:s3tables:REGION:ACCOUNT-ID:bucket/aws-s3/table/*"
    ]
  }]
}
```

#### For Querying Metadata Tables:

```json
{
  "Version": "2012-10-17",
  "Statement": [{
    "Sid": "PermissionsToQueryMetadataTables",
    "Effect": "Allow",
    "Action": [
      "s3tables:GetTable",
      "s3tables:GetTableData",
      "s3tables:GetTableMetadataLocation",
      "kms:Decrypt"
    ],
    "Resource": [
      "arn:aws:s3tables:REGION:ACCOUNT-ID:bucket/aws-s3",
      "arn:aws:s3tables:REGION:ACCOUNT-ID:bucket/aws-s3/table/*"
    ]
  }]
}
```

#### For SSE-KMS Encryption (if using):

**User/Role needs:**
- `s3tables:PutTableEncryption`
- `kms:DescribeKey`

**KMS Key Policy must grant:**
```json
{
  "Sid": "AllowS3MetadataAccess",
  "Effect": "Allow",
  "Principal": {
    "Service": [
      "metadata.s3.amazonaws.com",
      "maintenance.s3tables.amazonaws.com"
    ]
  },
  "Action": [
    "kms:GenerateDataKey",
    "kms:Decrypt"
  ],
  "Resource": "*"
}
```

## Metadata Table Types

### Journal Table (Required)
- **Purpose**: Captures change events (uploads, updates, deletes) in near real-time
- **Behavior**: Records only modifications occurring after configuration creation
- **Status**: Becomes "Active" within minutes
- **Storage**: Optional record expiration (7-2147483647 days) to reduce costs

### Live Inventory Table (Optional)
- **Purpose**: Provides complete, queryable inventory of all bucket objects and versions
- **Behavior**: Scans entire bucket to backfill existing object metadata
- **Status**: "Backfilling" ’ "Active" (minimum 15 minutes, can take hours for large buckets)
- **Use Cases**: Identify objects by storage class, tags, encryption status, etc.
- **Updates**: Reflects changes within one hour after backfilling completes

## Metadata Fields Available

Both tables capture:
- **System-defined metadata**: Creation time, storage class, size, version_id
- **Custom metadata**: Object tags, user-defined metadata from uploads
- **Event metadata**: Timestamps, requester details, record_type
- **Security metadata**: encryption_status, kms_key_arn
- **Additional fields**: key, sequence_number, user_metadata

## Provisioning Steps

### Step 1: Create Metadata Configuration File

Create a JSON configuration file (e.g., `metadata-config.json`):

**Minimal Configuration (Journal only, SSE-S3):**
```json
{
  "JournalTableConfiguration": {
    "EncryptionConfiguration": {
      "SseAlgorithm": "AES256"
    }
  }
}
```

**Full Configuration (Journal + Inventory, SSE-KMS, Record Expiration):**
```json
{
  "JournalTableConfiguration": {
    "RecordExpiration": {
      "Expiration": "ENABLED",
      "Days": 90
    },
    "EncryptionConfiguration": {
      "SseAlgorithm": "aws:kms",
      "KmsKeyArn": "arn:aws:kms:REGION:ACCOUNT-ID:key/KEY-ID"
    }
  },
  "InventoryTableConfiguration": {
    "ConfigurationState": "ENABLED",
    "EncryptionConfiguration": {
      "SseAlgorithm": "aws:kms",
      "KmsKeyArn": "arn:aws:kms:REGION:ACCOUNT-ID:key/KEY-ID"
    }
  }
}
```

**Configuration Parameters:**
- `RecordExpiration.Expiration`: "ENABLED" or "DISABLED"
- `RecordExpiration.Days`: Retention period (minimum 7 days)
- `SseAlgorithm`: "AES256" (default) or "aws:kms"
- `KmsKeyArn`: Required if using SSE-KMS
- `InventoryTableConfiguration.ConfigurationState`: "ENABLED" or "DISABLED"

### Step 2: Apply Configuration via AWS CLI

```bash
aws s3api create-bucket-metadata-configuration \
  --bucket YOUR-BUCKET-NAME \
  --metadata-configuration file://./metadata-config.json \
  --region REGION-CODE
```

### Step 3: Verify Configuration

```bash
aws s3api get-bucket-metadata-configuration \
  --bucket YOUR-BUCKET-NAME \
  --region REGION-CODE
```

### Step 4: Wait for Table Activation

**Journal Table:**
- Status changes to "Active" within a few minutes
- Begins capturing new events immediately

**Live Inventory Table (if enabled):**
- Status: "Backfilling" (minimum 15 minutes to hours)
- Progress depends on object count
- Once "Active", updates reflect within one hour

### Step 5: Grant Lake Formation Permissions

To query tables in Athena, grant Lake Formation permissions:

1. Open AWS Lake Formation console
2. Navigate to **Data lake permissions**
3. Click **Grant**
4. Select IAM users/roles that will run queries
5. Grant **Select** permissions on:
   - Database: `aws_s3_metadata`
   - Tables: Your metadata tables

### Step 6: Query with Athena

#### Access Tables

Tables are accessible via:
- **Catalog**: `s3tablescatalog/aws-s3`
- **Database**: `aws_s3_metadata`
- **Table naming**: Use all lowercase letters (capitals cause validation errors)

#### Example Queries

**Find objects by pattern:**
```sql
SELECT key, size, last_modified_date
FROM "s3tablescatalog/aws-s3"."aws_s3_metadata"."YOUR_TABLE_NAME"
WHERE key LIKE '%.jpg'
LIMIT 100;
```

**Track recent deletions:**
```sql
SELECT key, record_timestamp, requester
FROM "s3tablescatalog/aws-s3"."aws_s3_metadata"."YOUR_JOURNAL_TABLE"
WHERE record_type = 'DELETE'
  AND record_timestamp > current_timestamp - interval '7' day
ORDER BY record_timestamp DESC;
```

**Find unencrypted objects:**
```sql
SELECT key, encryption_status, storage_class
FROM "s3tablescatalog/aws-s3"."aws_s3_metadata"."YOUR_INVENTORY_TABLE"
WHERE encryption_status IS NULL
   OR encryption_status != 'SSE-KMS';
```

**Query by tags:**
```sql
SELECT key, object_tags, size
FROM "s3tablescatalog/aws-s3"."aws_s3_metadata"."YOUR_INVENTORY_TABLE"
WHERE object_tags['Environment'] = 'Production'
  AND object_tags['DataType'] = 'UserData';
```

**Recent changes (last 24 hours):**
```sql
SELECT key, record_type, record_timestamp
FROM "s3tablescatalog/aws-s3"."aws_s3_metadata"."YOUR_JOURNAL_TABLE"
WHERE record_timestamp > current_timestamp - interval '1' day
ORDER BY record_timestamp DESC;
```

## Storage and Billing

### Storage Location
- All metadata tables stored in AWS-managed table bucket named `aws-s3`
- One bucket per account per region
- Same region as source bucket

### Costs
- **Backfilling charges**: One-time fee for initial inventory table scan
- **Monthly fees**: For live inventory tables on buckets with >1 billion objects
- **Storage**: Standard S3 Tables storage pricing applies
- **Queries**: Athena query pricing based on data scanned

See [Amazon S3 Pricing](https://aws.amazon.com/s3/pricing/) for current rates.

## Important Limitations

1. **Encryption**: Cannot be modified after table creation
2. **Region**: Must match source bucket region
3. **Bucket level only**: Cannot apply to specific prefixes
4. **Table naming**: Must use all lowercase letters
5. **Record expiration**: Minimum 7 days if enabled
6. **Pre-July 2025 configurations**: AWS recommends recreating to access new features (record expiration, live inventory)

## Troubleshooting

### Common Issues

**Tables not visible in Athena:**
- Verify table names use all lowercase letters
- Check Lake Formation permissions granted
- Confirm analytics integration enabled for table bucket

**Insufficient permissions error:**
- Grant necessary Lake Formation permissions on specific table
- Verify IAM policy includes required s3tables actions

**Iceberg access error:**
- Grant Lake Formation permissions on table bucket catalog
- Grant permissions on database namespace
- Don't specify table-level grants for this error

**Backfilling takes too long:**
- Expected for large buckets (millions of objects)
- Monitor status via AWS Console or GetBucketMetadataConfiguration API
- No action required; process completes automatically

## Validation Checklist

After provisioning, verify:

- [ ] Configuration created successfully (GetBucketMetadataConfiguration returns details)
- [ ] Journal table status is "Active"
- [ ] Inventory table status is "Active" (if enabled, after backfilling completes)
- [ ] Lake Formation permissions granted to relevant users/roles
- [ ] Test query runs successfully in Athena
- [ ] Metadata updates reflect in queries (test with new object upload)
