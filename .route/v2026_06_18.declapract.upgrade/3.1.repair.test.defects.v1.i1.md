# 3.1.repair.test.defects

## defect inventory

---

### defect 1: Error cause type incompatibility

**what**: `jest.integration.env.ts:105` uses `new Error(msg, { cause })` which requires ES2022

**how**: declapract added `{ cause: error }` pattern to error throws

**why**: tsconfig targets es2020 but `Error.cause` was added in ES2022

**fix**: added `ES2022` to tsconfig.json lib array

**status**: fixed

---

### defect 2: env var name mismatch

**what**: `jest.unit.env.ts` used `STAGE` and `I_KNOW_WHAT_IM_DOING` env vars

**how**: the prior local jest.unit.env.ts used old env var names that predate the declapract standardization

**why**: declapract standardized on `CONFIG` and `I_KNOW_THE_RISKS` across jest env files

**fix**: aligned jest.unit.env.ts to use `CONFIG` and `I_KNOW_THE_RISKS`

**status**: fixed

---

### defect 3: dpdm cycles check finds node_modules cycles

**what**: lint fails due to 21 circular dependencies found in `node_modules/`

**how**: declapract added `test:lint:cycles` command that runs dpdm with `--transform` which follows imports

**why**: dpdm follows imports into node_modules and finds internal cycles in external packages

**cycles found**:
- ehmpathy packages: domain-objects, helpful-errors, with-simple-cache, rhachet, declastruct-aws
- third-party: bottleneck (Bottleneck.js <-> Group.js), aws-sdk (auth provider cycles)

**fix**:
1. upgraded all ehmpathy packages to latest (already at latest)
2. added exclusions to `.dpdmrc.yaml`: domain-objects, helpful-errors, with-simple-cache, rhachet, declastruct-aws, bottleneck, aws-sdk

**rationale**: cycles are internal to packages, not cycles we create. cannot fix package internals. enumerate each package instead of blanket node_modules exclusion.

**status**: fixed
