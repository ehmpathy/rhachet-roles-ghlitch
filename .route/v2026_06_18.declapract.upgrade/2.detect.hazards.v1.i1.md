## test hazards

### hazard 1: error cause type incompatibility

**file**: `jest.integration.env.ts:105`

**what changed**: the file uses `new Error(message, { cause: error })` syntax

**why it may break**: tsconfig targets `es2020`, but `Error.cause` was added in ES2022. typescript reports TS2554: Expected 0-1 arguments, but got 2.

**how to verify**:
```sh
rhx git.repo.test --what types
```

**fix options**:
- upgrade tsconfig target to `es2022` (may have downstream effects)
- remove `{ cause: error }` and use error wrap pattern instead

---

### hazard 2: env var name mismatch in jest.unit.env.ts

**file**: `jest.unit.env.ts`

**what changed**: `jest.integration.env.ts` uses `CONFIG` and `I_KNOW_THE_RISKS` env vars, but `jest.unit.env.ts` still uses the old names `STAGE` and `I_KNOW_WHAT_IM_DOING`

**why it may break**: inconsistent env var names between unit and integration test environments. may cause confusion or incorrect test isolation.

**how to verify**:
```sh
diff jest.unit.env.ts jest.integration.env.ts
```

**fix options**:
- align `jest.unit.env.ts` to use `CONFIG` and `I_KNOW_THE_RISKS`
