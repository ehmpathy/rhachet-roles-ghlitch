## notes / risks

- **tunnel persistence across composition**: `use.rds.capacity` calls `use.vpc.tunnel`
  as a subprocess, then polls with `pg_isready` after it returns. this relies on the ssm
  tunnel persisting beyond the `declastruct apply` process exit — which the prior design
  already relied on (pg_isready ran after `declastruct apply` returned in the same
  skill). so the composition preserves that assumption; verifiable only against real aws.
- `use.vpc.tunnel.ts` (declastruct engine) left unchanged — still reads `VPC_TUNNEL_*`;
  its `DECLASTRUCT_CONFIG` file path is now unused by the wrapper but kept intact (engine
  contract, harmless, aligns with keeping `--config` as a possible future).
