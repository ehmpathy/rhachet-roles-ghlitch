# 5.1 execution — yield

## what shipped

two per-verb prod-gate meters sharing one engine: `deploy.uses` and `provision.uses`.

- **engine** (`uses.operations.sh`) — three-scope cascade (global > org > local), env-keyed
  local grants, quota decrement + auto-revoke, TTY human guard, the `decide_uses` decision.
- **scope handlers** — `uses.local.sh` (set/allow/block/del/get), `uses.org.sh`
  (allow/block/del/get, block-only freeze policy), `uses.global.sh` (block/allow/get).
- **dispatcher** (`uses.sh`) + thin wrappers (`deploy.uses.sh`, `provision.uses.sh`).
- **gate** (`uses.check.sh`) — consumers call before a prod write; exit 0 = proceed, 2 = blocked.
- **gated consumers** — `deploy.sh`, `aws.cloudformation.rollback.sh` (deploy.uses);
  `provision.database.sh` apply-path, `provision.terraform.sh` writer-path (provision.uses).
- **the wisher's rule** — org `allow` only lifts a freeze; only a LOCAL grant permits prod.
- **tests** — `uses.integration.test.ts`, 24 green (real shell, no mocks, HOME-isolated state).

### fixed — my new engine code (iteration 2)

### non-issue — my code, defensible

- `read_all_org_states` / `uses.local get` use a single `jq 'to_entries | map(...) | .[]'`
  pipeline *inside a named read function*. that is the shell-level named-transformer pattern
  (jq is the transform language in bash), and it matches the reference meter engines the vision
  modeled (`radio.uses`, `git.commit.uses`). a bash wrapper around one jq line would add
  indirection without clarity, so left as-is.

### fixed — my new engine code (iteration 1, 5)

a corrupt meter state file must never read as a silent default — for a prod gate, a corrupt
global-freeze file silently read as "not blocked" would lift the freeze unseen. fixed:

- **fail loud on a corrupt state file** — added `assert_meter_file_valid` (top-level, so its
  `exit 1` propagates; an `exit` inside the read functions only exits the `$()` subshell and was
  unreliable). `uses.check.sh` and every `get` handler validate global/org/local upfront. a
  corrupt gate file now exits 1 (malfunction), not a silent allow/block. covered by `case10`.
- **org-freeze bypass closed** — `uses.check.sh` now fails loud if org policy is configured but
  the repo's org is unreadable (previously `get_org_from_keyrack`'s failure was ignored, which
  could drop an org freeze). org policy is consulted only when an org file is present.
- **atomic writes** — `write_local_uses` / `write_org_state` jq → temp then `mv`, so a corrupt
  base never truncates the live state file mid-write.
- the read functions dropped `2>/dev/null` (corruption is now caught loud upfront; `// default`
  still covers a valid file that omits a key).
