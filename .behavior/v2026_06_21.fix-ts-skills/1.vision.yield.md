# vision: fix ts skills

## the outcome world

### day-in-the-life

a mechanic runs `rhx query.database --env prod --sql "SELECT * FROM job LIMIT 5"`. the shell skill dispatches to the collocated TypeScript file via `exec npx tsx "$SCRIPT_DIR/query.database.ts"`. it just works — no path resolution errors, no "file not found".

### before/after

| before | after |
|--------|-------|
| shell skill symlinked to .agent/ | shell skill symlinked to .agent/ |
| collocated .ts file NOT symlinked | collocated .ts file also symlinked |
| `$SCRIPT_DIR` resolves to .agent/ | `$SCRIPT_DIR` resolves to .agent/ |
| .ts file not found → error | .ts file found → works |

### aha moment

the fix is simple: bundle .ts files in the npm package AND symlink them into .agent/. two changes, two repos, one working system.

## user experience

### usecases

- run `rhx query.database --env prod --sql "..."` to query the database
- run `rhx query.cloudwatch.logs --env prod --lambda createJob` to query logs
- shell skills dispatch to TypeScript for complex logic

### contract

shell skill (.sh) dispatches to collocated TypeScript (.ts) via:

```bash
exec npx tsx "$SCRIPT_DIR/query.database.ts" "$@"
```

### timeline

1. user runs `rhx query.database --env prod --sql "..."`
2. rhachet resolves skill from .agent/repo=ghlitch/role=observer/skills/query.database.sh
3. shell skill runs, `$SCRIPT_DIR` resolves to .agent/...
4. `exec npx tsx "$SCRIPT_DIR/query.database.ts"` finds the .ts file
5. TypeScript runs, connects to database, returns results

## mental model

### how users describe it

"shell skills can dispatch to TypeScript files. both file types live together and both get symlinked."

### analogy

like a shell command that sources a helper — both files need to be in the same place.

### terms

| we say | users say |
|--------|-----------|
| collocated .ts file | TypeScript implementation |
| dispatch | call, run |
| symlink | link, reference |

## evaluation

### how well does it solve the goal?

perfectly. shell skills can dispatch to TypeScript and it just works.

### pros

- simple fix: add .ts to rsync includes + symlink patterns
- no new patterns or complexity
- works with extant npx tsx dispatch pattern

### cons

- requires coordinated change in two repos (ghlitch + rhachet)
- rhachet#393 must land before this fix is complete

### verification requirement

an acceptance test must prove the fix works end-to-end:
1. use genTempDir() from test-fns to create isolated environment
2. install the package via npm
3. run `npx rhachet init --roles ghlitch`
4. invoke a skill with .ts dispatch (e.g., `rhx query.database --help`)
5. verify it succeeds without "file not found" error

this is the most important requirement — without it, we assume the fix works without proof.

### edge cases

| edge case | handling |
|-----------|----------|
| .ts file without .sh wrapper | just gets symlinked, no harm |
| .sh file without .ts | works as before |
| nested .ts imports | work because both files are collocated |
