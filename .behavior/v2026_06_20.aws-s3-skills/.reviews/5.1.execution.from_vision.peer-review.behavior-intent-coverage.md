# blocker.1: Missing snapshot coverage for alternative help invocation methods

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.verification/rule.require.contract-snapshot-exhaustiveness.md

**locations**:
- src/domain.roles/observer/skills/aws.s3.get.integration.test.ts:case2 and case3
- src/domain.roles/observer/skills/aws.s3.list.integration.test.ts:case2 and case3

The integration tests for aws.s3.get and aws.s3.list cover the --help flag with a snapshot, but they do not include snapshots for the '-h' flag and the 'help' positional argument. According to rule.require.contract-snapshot-exhaustiveness, every CLI command must have exhaustive snapshot coverage for all output variants. The '-h' and 'help' invocations are distinct ways to trigger help output and could drift from the '--help' output over time. Without snapshots, regressions in these alternative help paths will go undetected.

Both test files contain test cases for case2 (-h) and case3 (help positional) that only check partial output (stdout includes 'heres the deal') but lack a snapshot assertion. A snapshot should be added to each of these test cases.

**snippet**:
```ts
// In aws.s3.get.integration.test.ts, case2:
  given('[case2] -h flag', () => {
    when('[t0] -h is passed', () => {
      const result = useThen('skill runs', () => runSkill('-h'));

      then('it exits 0', () => {
        expect(result.exitCode).toBe(0);
      });

      then('it shows help output', () => {
        expect(result.stdout).toContain('heres the deal');
      });
      // Missing: then('help output matches snapshot', () => { ... })
    });
  });

// Similar pattern in aws.s3.list.integration.test.ts case2 and case3
```
