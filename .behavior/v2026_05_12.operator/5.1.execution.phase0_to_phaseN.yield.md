## blockers: peer review findings (21 blockers)

### issue: review scope mismatch

peer reviews scoped to `**/*.ts` missed the primary `.sh` implementations:
- behavior-intent-coverage flagged "absent skills" but `.sh` files exist for all skills
- the `.ts` files are implementation details called by `.sh` wrappers

### legitimate code quality issues in `.ts` files

the TypeScript implementation files have code quality issues:

**query.database.ts** (observer):
- redundant arg parse (already done in .sh wrapper)
- failhide pattern (catches all errors, logs only message)
- mutable variables (let, array.push)
- else-if branches instead of early returns
- no --help handler (handled in .sh but .ts re-parses args)

**use.vpc.tunnel.ts** (supporter):
- polyglot header (#!/bin/bash + TypeScript) flagged as friction
- plain Error instead of ConstraintError
- no validation when config loaded from file
- mutable array (resources.push)

### options to proceed

1. **fix all code quality issues** — significant refactor of .ts files
2. **simplify .ts files** — move arg parse entirely to .sh, pass via env vars
3. **accept as-is** — document as tech debt, these skills work in svc-jobs
4. **remove .ts files** — reimplement in bash (complex for db queries)

### resolution

**accepted option 3**: accept as-is with tech debt.

- skills work in production (svc-jobs)
- .sh files are the public contract and are solid
- .ts code quality issues captured in dream: `.dream/2026_06_16.rearchitect-ghlitch-ts-skills.dream.md`
