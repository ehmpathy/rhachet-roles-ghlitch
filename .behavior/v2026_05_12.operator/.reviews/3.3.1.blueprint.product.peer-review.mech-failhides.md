# nitpick.1: Constraint error uses exit 1 instead of 2

**rule**: .agent/repo=ehmpathy/role=mechanic/briefs/practices/code.prod/pitofsuccess.errors/rule.require.exit-code-semantics.md

**locations**:
- .behavior/v2026_05_12.operator/3.3.1.blueprint.product.yield.md

In the generalization requirements section, the env validation implementation uses exit 1 for invalid --env input. This is a constraint error (bad user input that the caller must fix before retrying). Per the rule, constraint errors must use exit code 2 while exit 1 is reserved exclusively for malfunctions (external errors like AWS failures). Using exit 1 here will mislead callers about whether to retry or fix input, and conflicts with the exit code guarantees documented in the skill headers within the same file. The blueprint must correct this pattern before implementation.

**snippet**:
```sh
if [[ ! "$ENV" =~ ^(test|prep|prod)$ ]]; then
  echo "error: --env must be test, prep, or prod" >&2
  exit 1
fi
```
