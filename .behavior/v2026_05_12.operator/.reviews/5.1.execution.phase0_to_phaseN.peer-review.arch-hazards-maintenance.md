# blocker.1: Mutable loop variable and in-place mutation

**rule**: .agent/repo=ehmpathy/role=mechanic/briefs/practices/code.prod/pitofsuccess.procedures/rule.require.immutable-vars.md.min

**locations**:
- src/domain.roles/observer/skills/query.database.ts:27

The argument parsing code declares a loop index with `let` and mutates it using `i++` and `args[++i]` to skip over flag values. It also declares `sql` and `format` with `let` and reassigns them, plus mutates a `chunks` array via `.push()`. This violates the immutability rule which requires `const` for all bindings and forbids any mutation (use spread/clone/reduce instead). Mutable state is explicitly called out as a maintenance hazard in the forbid.maintenance-hazards rule because it hides errors, creates race conditions, and makes code hard to reason about.

**snippet**:
```ts
  const args = process.argv.slice(2);
  let sql = '';
  let format = 'table';

  for (let i = 0; i < args.length; i++) {

      const value = args[++i];

        }
        sql = Buffer.concat(chunks).toString('utf-8').trim();
      } else {
        sql = value ?? '';
      }
    } else if (arg === '--format') {
      format = args[++i] ?? 'table';
    } else if (
      ['--env', '--limit', '--repo', '--role', '--skill'].includes(arg)
    ) {
      i++; // skip value
```

# blocker.4: Error swallowing in top-level catch handler

**rule**: .agent/repo=ehmpathy/role=mechanic/briefs/practices/code.prod/pitofsuccess.errors/rule.forbid.failhide.md.pt1.md.min

**locations**:
- src/domain.roles/observer/skills/query.database.ts:93

The `main().catch` handler catches every error, logs only a generic prefix plus `error.message`, and calls `process.exit(1)` without rethrowing or preserving stack/cause/context. This is a direct failhide violation. The rule requires that catch blocks only allowlist expected errors and must rethrow all unexpected errors; silent swallowing is forbidden as it hides real errors, causes production defects, and wastes debug hours. This is a blocker per the maintenance-hazards rule.

**snippet**:
```ts
main().catch((error) => {
  console.error('🔮 path obscured...');
  console.error('');
  console.error(`   ${error.message}`);
  process.exit(1);
});
```
