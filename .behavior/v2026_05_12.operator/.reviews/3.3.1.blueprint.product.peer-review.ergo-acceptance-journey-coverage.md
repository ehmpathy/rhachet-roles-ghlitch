# blocker.1: Acceptance tests for new CLI skills planned with fakes instead of real external contract calls

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/ergonomist/rule.require.acceptance-journey-coverage.md

**locations**:
- .behavior/v2026_05_12.operator/3.3.1.blueprint.product.yield.md

The blueprint introduces 14 new user-faced CLI skills (contracts) across observer, operator, and deployer roles. Per the rule, every user journey requires acceptance tests exercising real user journeys with given/when/then, plus snapshot coverage for positive/negative/edge paths. For external contracts, integration/acceptance tests must use real api calls (not mocks), and credential difficulty is a blocker not deferral. The target instead describes a tiered strategy relying on fake aws CLI via PATH injection (tier 2) with canned responses as fixtures, and only real AWS calls if AWS_ACCOUNT_ID is present (otherwise throw ConstraintError). It explicitly labels tier 3 as 'documented exception' and defers full real verification to quarterly runs on another account. This means the planned blackbox/*.acceptance.test.ts and *.integration.test.ts will not prove the real contracts (AWS, RDS, SSM, CloudWatch) work, violating the requirement for real calls and exhaustive coverage without gaps or excuses.

**snippet**:
```ts
/**
 * .mock = none (real AWS required)
 * .why = RDS connection, SSM tunnel, CloudWatch queries need real infra
 * .real = manual verification documented below
 * .quarterly = run full suite against ahbode/svc-jobs test account
 */
if (!process.env.AWS_ACCOUNT_ID) {
  throw new ConstraintError('AWS_ACCOUNT_ID required', {
    hint: 'run against ahbode/svc-jobs test account or provision test resources',
    doc: 'docs/test-aws-skills.md',
  });
}
```
