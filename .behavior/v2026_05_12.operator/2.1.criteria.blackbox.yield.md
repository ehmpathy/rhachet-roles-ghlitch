# blackbox criteria: promote generic skills from svc-jobs

---

## episode experience

### usecase.1 = investigate production incident

```
given(user needs to understand a production issue)
  when(user queries cloudwatch metrics)
    then(metrics show error counts sorted by severity)
      sothat(user identifies which lambda fails)
  when(user queries cloudwatch logs for that lambda)
    then(logs show error messages with timestamps)
      sothat(user identifies error pattern)
  when(user queries database for stuck records)
    then(query results show affected rows)
      sothat(user confirms scope of issue)
```

### usecase.2 = verify data state

```
given(user needs to check database records)
  when(user runs a readonly query)
    then(results display in specified format)
      sothat(user sees record state)
  when(user attempts a write query)
    then(query is rejected with clear error)
      sothat(user cannot accidentally modify prod data)
```

### usecase.3 = run maintenance task

```
given(user needs to execute an operational command)
  when(user lists available commands)
    then(all registered commands display with descriptions)
      sothat(user knows what commands exist)
  when(user invokes a command with arguments)
    then(command executes and output streams to stdout)
      sothat(user sees result immediately)
  when(user invokes command against wrong env)
    then(explicit env flag prevents accidental execution)
      sothat(user cannot hit wrong environment)
```

### usecase.4 = check system health

```
given(user needs to verify system vitals)
  when(user lists available vitals)
    then(all health checks display)
      sothat(user knows what to check)
  when(user invokes a vital)
    then(health status returns with details)
      sothat(user knows if system is healthy)
  when(vital fails)
    then(failure details explain what is wrong)
      sothat(user can diagnose the issue)
```

### usecase.5 = deploy service

```
given(user needs to push code to production)
  when(user runs deploy for an environment)
    then(deployment logs stream to stdout)
      sothat(user sees progress in real-time)
  when(deploy succeeds)
    then(success confirmation displays)
      sothat(user knows deployment completed)
  when(deploy fails)
    then(failure reason surfaces with next steps)
      sothat(user knows how to recover)
```

### usecase.6 = diagnose deploy failure

```
given(deploy has failed and user needs to understand why)
  when(user checks cloudformation status)
    then(stack state displays with recent events)
      sothat(user sees what failed)
  when(stack shows failed resources)
    then(failure reasons display for each resource)
      sothat(user identifies root cause)
```

### usecase.7 = recover stuck deploy

```
given(cloudformation stack is stuck in rollback)
  when(user triggers rollback continuation)
    then(rollback progress streams to stdout)
      sothat(user sees recovery progress)
  when(rollback completes)
    then(stack returns to stable state)
      sothat(user can retry deployment)
```

### usecase.8 = manual tunnel setup

```
given(user needs direct database access)
  when(user starts rds capacity tunnel)
    then(tunnel activates and database endpoint available at localhost)
      sothat(user can run psql directly)
  when(user exits the skill)
    then(tunnel closes and resources clean up)
      sothat(no orphan connections remain)
```

### usecase.9 = check ssm parameters

```
given(user needs to verify config in ssm)
  when(user checks a parameter value)
    then(current value displays)
      sothat(user sees what is configured)
  when(user provides expected value)
    then(match/mismatch result returns)
      sothat(user validates config correctness)
```

### usecase.10 = provision database

```
given(user needs to create or update rds instance)
  when(user provisions database for an environment)
    then(provisioned endpoint returns)
      sothat(user has connection details)
  when(user specifies --wait flag)
    then(skill waits until database is ready)
      sothat(user gets immediate confirmation of availability)
```

### usecase.11 = manage infrastructure via terraform

```
given(user needs to modify infrastructure)
  when(user runs terraform plan)
    then(planned changes display)
      sothat(user reviews before apply)
  when(user runs terraform apply)
    then(changes execute and output streams)
      sothat(user sees infrastructure updates)
  when(user targets specific resource)
    then(only that resource is affected)
      sothat(user has precise control)
```

### usecase.12 = local test database

```
given(user needs a local database for tests)
  when(user starts testdb)
    then(database container starts on specified port)
      sothat(user has local db for tests)
  when(user checks testdb status)
    then(current state displays with connection string)
      sothat(user can connect)
  when(user stops testdb)
    then(container stops and port frees)
      sothat(resources release)
```

## exchange experience

### query.cloudwatch.logs

```
given(valid env and optional filters)
  when(skill invoked)
    then(logs stream to stdout)
    then(summary stats write to cache file)

given(invalid env)
  when(skill invoked)
    then(error: env must be prep or prod)

given(--tail flag)
  when(skill invoked)
    then(logs follow in real-time until interrupted)
```

### query.cloudwatch.metrics

```
given(valid env)
  when(skill invoked)
    then(metrics display sorted by count desc)

given(--lambda filter)
  when(skill invoked)
    then(only metrics for that lambda show)

given(--namespace sqs)
  when(skill invoked)
    then(sqs metrics display instead of lambda)
```

### query.database

```
given(valid env and sql query)
  when(skill invoked)
    then(results display in specified format)

given(write query attempted)
  when(skill invoked)
    then(error: readonly connection rejects INSERT/UPDATE/DELETE/DROP)

given(--format csv)
  when(skill invoked)
    then(output is csv format)

given(--format json)
  when(skill invoked)
    then(output is json format)

given(query exceeds --limit)
  when(skill invoked)
    then(results truncate at limit with warn message)
```

### aws.ssm.param.check

```
given(valid env and param path)
  when(skill invoked)
    then(parameter value displays)

given(--expected value matches)
  when(skill invoked)
    then(exit 0)

given(--expected value differs)
  when(skill invoked)
    then(exit 1 with mismatch details)

given(param does not exist)
  when(skill invoked)
    then(error: parameter not found)
```

### invoke.command

```
given(valid env and command name)
  when(skill invoked)
    then(command executes and output streams)

given(--list flag)
  when(skill invoked)
    then(available commands display with descriptions)

given(command does not exist)
  when(skill invoked)
    then(error: command not found with available commands listed)

given(absent --env)
  when(skill invoked)
    then(error: --env required)
```

### invoke.vital

```
given(valid env and vital name)
  when(skill invoked)
    then(vital executes and health status returns)

given(--list flag)
  when(skill invoked)
    then(available vitals display)

given(--alert flag and vital fails)
  when(skill invoked)
    then(alert triggered in addition to output)

given(vital does not exist)
  when(skill invoked)
    then(error: vital not found)
```

### deploy

given(valid env)

given(deploy succeeds)

given(deploy fails)

### aws.cloudformation.status

```
given(valid env)
  when(skill invoked)
    then(stack status displays)
    then(recent failed events display with reasons)

given(stack does not exist)
  when(skill invoked)
    then(error: stack not found)
```

### aws.cloudformation.rollback

```
given(stack in UPDATE_ROLLBACK_FAILED state)
  when(skill invoked)
    then(rollback continues and progress streams)
    then(waits for completion)

given(stack not in rollback state)
  when(skill invoked)
    then(error: stack not in rollback state)
```

### provision.database

```
given(valid env)
  when(skill invoked)
    then(database provisioned and endpoint returns)

given(--wait flag)
  when(skill invoked)
    then(waits until db ready before return)

given(database already exists)
  when(skill invoked)
    then(idempotent: returns extant endpoint)
```

### provision.terraform

```
given(valid env and --action plan)
  when(skill invoked)
    then(terraform plan output displays)

given(valid env and --action apply)
  when(skill invoked)
    then(terraform apply output streams)

given(--target specified)
  when(skill invoked)
    then(only targeted resource affected)

given(invalid env)
  when(skill invoked)
    then(error: env must be prep or prod)
```

### use.rds.capacity

```
given(valid env)
  when(skill invoked)
    then(tunnel starts and db available at localhost)
    then(tunnel remains active until skill exits)

given(db in serverless pause state)
  when(skill invoked)
    then(waits up to 180s for db to wake)
    then(error if db never wakes)

given(keyrack not unlocked)
  when(skill invoked)
    then(error: keyrack unlock required)
```

### use.vpc.tunnel

```
given(valid env and target)
  when(skill invoked)
    then(ssm tunnel starts)
    then(local port forwards to target)

given(sudo not available)
  when(skill invoked with hosts modification)
    then(error: sudo required for /etc/hosts)
```

### use.testdb

```
given(start subcommand)
  when(skill invoked)
    then(database container starts)
    then(connection string outputs)

given(status subcommand)
  when(skill invoked)
    then(current state displays)

given(stop subcommand)
  when(skill invoked)
    then(container stops)
    then(port frees)

given(start when already active)
  when(skill invoked)
    then(idempotent: returns extant connection)
```

## shared constraints

```
given(any skill that requires keyrack)
  when(keyrack not unlocked)
    then(clear error with setup instructions)

given(any skill with --env flag)
  when(--env not provided)
    then(error: --env is required)
    then(never default to any environment)

given(any skill with --env flag)
  when(--env invalid)
    then(error: env must be one of test, prep, prod)
```
