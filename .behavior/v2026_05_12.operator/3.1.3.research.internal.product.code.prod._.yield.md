## source repo: ahbode/svc-jobs

### pattern.1 = skill shell+typescript split

[1] `.agent/repo=.this/role=any/skills/show.db.query.sh` lines 1-91

skills follow a shell wrapper + typescript implementation pattern:

```
skill.sh → handles env parse, keyrack unlock, aws credential export → exec skill.ts
```

**key quotes:**

> ```bash
> # source aws credentials from keyrack
> AWS_PROFILE=$(rhx keyrack get --owner ehmpath --env "$ENV" --key AWS_PROFILE --value 2>/dev/null || echo "")
> ```
> — show.db.query.sh:74

> ```bash
> # export static credentials only — do NOT export AWS_PROFILE
> # AWS SDK prefers AWS_PROFILE over static creds, which causes SSO failures
> eval "$(aws configure export-credentials --profile "$AWS_PROFILE" --format env)"
> unset AWS_PROFILE AWS_DEFAULT_PROFILE
> ```
> — show.db.query.sh:82-84

> ```bash
> # run the TypeScript implementation
> exec npx tsx "$SKILL_DIR/show.db.query.ts" "$@"
> ```
> — show.db.query.sh:91

**verdict**: [REUSE] — this pattern works well for credentials + business logic separation
