# blocker: 5.3.verification

## what blocks me

the peer review system cannot find rule files because the glob library used by the review skill does not follow symlinks.

the rule files exist at `.agent/repo=ehmpathy/role=mechanic/briefs/practices/code.test/...` but these are symlinks to `node_modules/.pnpm/rhachet-roles-ehmpathy@.../dist/domain.roles/mechanic/briefs/...`.

| tool | finds files? |
|------|--------------|
| `cat .agent/repo=ehmpathy/role=mechanic/briefs/.../rule.md` | yes |
| `rhx globsafe --pattern '.agent/.../rule.md'` | yes |
| peer review skill glob | no ("rules glob found nada") |

this causes r7 to permanently fail with "blocked by constraints" even though the rule files exist and are accessible via direct file access.

## what i tried

1. **restored symlinks**: ran `npx rhachet init --roles mechanic` which linked 231 briefs, 83 skills, 20 inits
2. **deleted cached reviews**: removed stale `.route/5.3.verification.guard.review.*.md` files multiple times
3. **verified symlinks work**: confirmed via `cat` and `globsafe` that files are accessible
4. **multiple `--as arrived` attempts**: each regenerated the review with the same "rules glob found nada" error

the root cause: the review skill's glob library (likely Node.js glob or ripgrep) does not follow symlinks by default.

## what i need

one of these options:

1. **fix the review skill** to follow symlinks when it resolves rule file paths
2. **human approval** to proceed despite the r7 review failure (the actual code changes are complete and verified - 30 unit tests pass, all self-reviews pass, only r7 has this infrastructure bug)
3. **copy files instead of symlink** - modify rhachet init to copy files rather than create symlinks (less ideal for maintenance)

## evidence

```
$ rhx globsafe --pattern '.agent/repo=ehmpathy/role=mechanic/briefs/practices/code.test/scope.coverage/*.md'

$ # but peer review shows:

   hint: verify the glob pattern matches files in your repo  # same path, not found
```

## summary

infrastructure bug: peer review glob does not follow symlinks. human intervention required to approve or fix.
